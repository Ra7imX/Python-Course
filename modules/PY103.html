L'indentation en Python - Règles et importance

Objectifs d'apprentissage

  1. Comprendre le rôle crucial de l'indentation en Python
  2. Maîtriser les règles d'indentation
  3. Éviter les erreurs courantes liées à l'indentation
  4. Apprendre à structurer son code proprement

1. Principes de base de l'indentation

L'indentation en Python est un élément fondamental du langage :

  • Définit la structure du code
  • Détermine les blocs d'instructions
  • Utilise généralement 4 espaces (standard PEP 8)
  • Est OBLIGATOIRE (contrairement à d'autres langages)

Contrairement à d'autres langages qui utilisent des accolades {} ou des mots-clés comme "begin/end", Python utilise l'indentation pour définir les blocs de code. C'est une particularité qui rend le code Python naturellement lisible.

2. Exemples simples d'indentation

Exemple avec une structure conditionnelle :

ville = "Marrakech"
if ville == "Marrakech":
print("Bienvenue dans la ville rouge!") # Indenté avec 4 espaces
print("Visitez la place Jemaa el-Fna") # Même niveau d'indentation
print("Bon séjour au Maroc!") # Retour au niveau 0

Points importants :

  • Le bloc conditionnel commence après les deux-points (:)
  • Toutes les lignes du bloc doivent avoir la même indentation
  • Le retour à un niveau d'indentation inférieur indique la fin du bloc

3. Niveaux multiples d'indentation

Les structures imbriquées nécessitent plusieurs niveaux d'indentation :

def presenter_ville(ville, population):
# Premier niveau d'indentation
print(f"Information sur {ville}:")
if population > 1000000:
# Deuxième niveau d'indentation
print(f"{ville} est une grande ville")
if ville == "Casablanca":
# Troisième niveau d'indentation
print("C'est la plus grande ville du Maroc")
else:
# Retour au deuxième niveau
print(f"{ville} est une ville moyenne ou petite")

# Utilisation de la fonction
presenter_ville("Casablanca", 3400000)
presenter_ville("Ifrane", 35000)

Observations :

  • Chaque structure imbriquée ajoute un niveau d'indentation
  • L'indentation permet de visualiser clairement la structure du code
  • Les blocs alternatifs (comme else) maintiennent le même niveau que leur condition associée

4. Erreurs courantes d'indentation

Voici des exemples d'erreurs à éviter absolument :

1. Indentation incohérente :

if ville == "Fès":
print("Ville historique")
print("Médina classée") # ERREUR: Indentation différente

2. Oubli d'indentation :

if ville == "Tanger":
print("Ville du détroit") # ERREUR: Pas d'indentation

3. Mélange espaces et tabulations :

if ville == "Agadir":
print("Ville balnéaire") # Espaces
print("Beau climat") # Tabulation (à éviter)

Conséquences des erreurs d'indentation :

  • Erreurs de syntaxe (IndentationError, TabError)
  • Comportement inattendu du programme
  • Difficultés de débogage

5. Cas pratiques d'indentation

Exemple complet avec plusieurs niveaux d'indentation :

def gerer_commande_restaurant():
# Menu du restaurant
plats = {
"tajine": 70,
"couscous": 80,
"pastilla": 90
}

# Demande au client
commande = input("Que souhaitez-vous commander ? ")
if commande in plats:
print(f"Prix du {commande}: {plats[commande]} DH")
quantite = int(input("Combien en voulez-vous ? "))
if quantite > 0:
total = plats[commande] * quantite
if total > 200:
print("Vous avez droit à un thé gratuit!")
print(f"Total à payer: {total} DH")
else:
print("Quantité invalide")
else:
print("Désolé, ce plat n'est pas au menu")

Remarquez comment l'indentation :

  • Indique clairement la structure logique du programme
  • Permet de suivre le flux d'exécution
  • Facilite la compréhension des conditions imbriquées

6. Règles et bonnes pratiques

Règles d'indentation :

  • Toujours utiliser le même nombre d'espaces
  • 4 espaces est le standard Python (PEP 8)
  • Être cohérent dans tout le projet
  • Ne pas mélanger espaces et tabulations

Quand indenter :

  • Après les ':' dans if, for, while, def, class
  • Pour tous les blocs de code
  • Pour les lignes continuées
  • Dans les listes/dictionnaires multilignes

Outils et astuces :

  • Utiliser un éditeur qui montre les espaces
  • Configurer l'auto-indentation
  • Utiliser des outils de formatage (comme autopep8)
  • Vérifier l'indentation avant d'exécuter

7. Exercices pratiques

Exercices à réaliser :

  1. Structure simple :
    • Écrivez un programme qui vérifie l'âge pour un prix d'entrée
    • Utilisez plusieurs conditions imbriquées
    • Assurez-vous que l'indentation est correcte
  2. Correction d'erreurs :
    • Corrigez le code suivant :
      • def calculer_prix():
      prix = 100
      if prix > 50:
      print("Cher")
      else:
      print("Abordable")
  3. Structures imbriquées :
    • Créez un programme de réservation d'hôtel
    • Utilisez au moins 3 niveaux d'indentation
    • Gérez différents cas (disponibilité, prix, options)
  4. Pratique avancée :
    • Écrivez un programme de gestion de match de foot
    • Incluez plusieurs niveaux de décision
    • Utilisez des boucles et conditions imbriquées
  5. Défi d'indentation :
    • Créez une calculatrice de voyage au Maroc
    • Gérez différentes villes, moyens de transport
    • Utilisez des structures profondément imbriquées
