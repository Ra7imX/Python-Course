Les commentaires et la documentation du code Python

Objectifs d'apprentissage

  1. Comprendre l'importance des commentaires
  2. Maîtriser les différents types de commentaires
  3. Apprendre les bonnes pratiques de documentation
  4. Savoir quand et comment commenter son code

1. Types de commentaires en Python

Python offre plusieurs façons de commenter votre code :

  • Commentaires sur une ligne :
    • Commencent par le caractère # (dièse)
    • Peuvent être placés sur une ligne dédiée
    • Peuvent également être placés en fin de ligne
  • Commentaires multi-lignes :
    • Utilisent des triples guillemets doubles (""") ou simples (''')
    • Permettent d'écrire des commentaires sur plusieurs lignes
    • Souvent utilisés pour la documentation de fonctions (docstrings)

Exemples de commentaires :

# Ceci est un commentaire sur une ligne
print("Salam!") # Commentaire en fin de ligne

"""
Ceci est un commentaire
sur plusieurs lignes avec
des guillemets doubles
"""

'''
Autre façon de faire
des commentaires sur
plusieurs lignes avec
des guillemets simples
'''

2. Documentation de variables

Les commentaires permettent d'expliquer l'utilisation des variables :

# Variables pour un match de football
score_wydad = 2 # Buts marqués par le Wydad
score_raja = 1 # Buts marqués par le Raja
temps_jeu = 90 # Durée du match en minutes
est_termine = True # Indique si le match est fini

Ces commentaires permettent de :

  • Clarifier l'objectif de chaque variable
  • Préciser les unités utilisées
  • Documenter les valeurs attendues

3. Documentation de processus

Pour des calculs ou processus complexes, les commentaires aident à suivre la logique :

# Calculer le prix total d'un repas traditionnel
prix_plat = 75 # Prix du plat principal en DH
prix_dessert = 25 # Prix du dessert en DH
prix_the = 10 # Prix du thé à la menthe en DH

# Calcul du total avec TVA
tva = 0.20 # Taux de TVA à 20%
total_ht = prix_plat + prix_dessert + prix_the # Total hors taxe
total_ttc = total_ht * (1 + tva) # Total TTC

Avantages de cette approche :

  • Décomposition claire des étapes
  • Documentation des constantes et des formules
  • Facilité de maintenance du code

4. Documentation de fonctions

Les fonctions en Python doivent être documentées avec des "docstrings" :

def calculer_distance(ville1, ville2, vitesse):
"""
Calcule le temps de trajet entre deux villes marocaines.

Arguments :
ville1 (str) : Ville de départ
ville2 (str) : Ville d'arrivée
vitesse (float) : Vitesse moyenne en km/h

Retourne :
float : Temps de trajet en heures
"""
# Dictionnaire des distances entre villes (en km)
distances = {
("Casablanca", "Rabat"): 87,
("Rabat", "Tanger"): 250,
("Casablanca", "Marrakech"): 240
}

# Calcul du temps de trajet
distance = distances.get((ville1, ville2), 0)
return distance / vitesse if distance > 0 else 0

Une bonne docstring contient :

  • Description générale de la fonction
  • Paramètres avec leurs types et descriptions
  • Valeur de retour avec type et description
  • Exemples d'utilisation (facultatif)
  • Notes sur les cas particuliers (facultatif)

5. Commentaires pour le débogage

Les commentaires peuvent être utilisés pour le débogage temporaire :

# DEBUG: Afficher les valeurs pour vérification
# print(f"Distance: {distance} km")
# print(f"Vitesse: {vitesse} km/h")
# print(f"Temps: {temps} heures")

Pratiques de débogage :

  • Commenter le code de débogage plutôt que de le supprimer
  • Utiliser des préfixes comme DEBUG ou TODO
  • Se rappeler de nettoyer ces commentaires avant la mise en production

6. Bonnes pratiques de commentaires

Pour des commentaires efficaces, suivez ces conseils :

  • Quand commenter :
    • Pour expliquer le "pourquoi" plutôt que le "quoi"
    • Pour documenter des décisions importantes
    • Pour clarifier du code complexe
    • Pour indiquer des cas particuliers
  • Quand ne pas commenter :
    • Code évident ou auto-documenté
    • Informations redondantes
    • Code temporaire ou obsolète
    • Informations qui peuvent devenir obsolètes
  • Style de commentaires :
    • Rester concis et précis
    • Utiliser un langage professionnel
    • Maintenir les commentaires à jour
    • Être cohérent dans le style
  • Types de commentaires utiles :
    • Documentation de fonction (docstring)
    • Explications d'algorithmes complexes
    • Notes sur les limitations
    • TODO et FIXME pour le suivi

7. Exercices pratiques

Exercices à réaliser :

  1. Documentation de base :
    • Créez des variables pour décrire un match de foot
    • Commentez chaque variable de manière appropriée
    • Ajoutez un commentaire multiligne expliquant le contexte
  2. Documentation de calculs :
    • Écrivez un programme qui calcule le prix d'un voyage
    • Commentez chaque étape du calcul
    • Ajoutez des commentaires pour les cas particuliers
  3. Documentation de fonction :
    • Créez une fonction qui convertit des températures
    • Écrivez une docstring complète
    • Ajoutez des commentaires pour les cas limites
  4. Pratique des commentaires :
    • Prenez un code existant sans commentaires
    • Ajoutez des commentaires appropriés
    • Identifiez les parties qui n'ont pas besoin de commentaires
  5. Défi de documentation :
    • Documentez un petit programme de gestion de restaurant
    • Utilisez tous les types de commentaires
    • Suivez les bonnes pratiques de documentation
